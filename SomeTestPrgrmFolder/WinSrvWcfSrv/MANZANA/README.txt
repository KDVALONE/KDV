WcfDbEchoService
WCF Служба работающая в нутри Win службы. Работающая череp Dupper с БД
Реализован выбор БД с использованием разных репозиториев через файло конфигураций
В примере по умолчанию используется подменная версия FakeRepository 
- Служба реализует 2 метода
1 Сохранение в БД полученного Чека, в формате JSON, посредством вызова хранимой процедуры на стороне БД
2 Возвращает N последних уникальных чеков в случайном порядке. (тоесть, если ID последних N=3 чеков 101, 102, 103 то может вернуть Чеки с этим ID В случайном порядке 102, 103, 101 и тд)

FolderWatcherService
Win Cлужба наблюждающая за указанной папкой, читающая JSON файл, передающая на WCF службу 
Служба мониорит указанную папку Input (все путик к папкам могут меняться через конфиг или использоваться со значениями по умолчанию)
Если в папку Input попадает .txt файл, то он проверяется на соответсвие модели чека формата JSON, если содержание файла корректно
то данные чека передаются на WCF службу, сам файл перемещается в папку Complete,
если данные не удовлетворяют модели чека, или файл другого формата, или он является каталогом , то такой файл или каталог перемещаются в папку  Garbage
Перемещение каталогов учитывает вложенные в каталоги файлы, а так же тот факт что в папке Garbage может быть каталог или файл с таким же названием
Так же, служба принимает от WCF службы коллекцию указанных N чеков и сохраняет их в папку SaveCheques  (N - задается в файле конфигурации или исп. значение по умолчанию N=4)


В обеих службах реализованно логированние с помощью log4net

Установка служб осуществляется с помощью .bat файлов MyServiceDelete.bat и MyServiceInstaller.bat , запускать с правами Администратора

Взаимодействие двух служб между собой:
WcfDbEchoService - В роли Хоста, ждет обращение от FolderWatcherService
FolderWatcherService - В роли Клиента, обращается к WcfDbEchoService
